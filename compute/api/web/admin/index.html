MOG2 Tuning

↓ lower = detect more. MOG2's foreground sensitivity — lower lets subtler pixel changes count (dark / low-contrast night cats).
↓ lower = detect more. How fast a still object fades into the background — lower keeps a paused / lingering cat foreground longer.
↓ lower = detect more. Smallest blob that counts, as a fraction of the ROI — lower admits smaller / distant / partial-view cats.
↑ raise if close cats missed. Largest blob that counts — rejects whole-ROI light jumps. Raise only if a cat right under the camera fills the frame.
↓ lower = detect more. Consecutive motion frames before it fires; 1 catches fast / brief crossings. Counted in frames, not seconds — at higher fps the same value spans less time.
↑ raise if tiny blobs vanish. ROI width (px) fed to MOG2 — higher preserves small / distant blobs, at more CPU cost.

1. Build gallery

Embeds labelled, identified crops into a new versioned gallery (a draft model version). Defaults to gallery-grade crops only — protect the gallery; widen to ok/poor to A/B whether crop quality is the bottleneck. Building alone changes nothing live — Promote below makes a version active.

2. Validate

Runs the feasibility probe over labelled, identified crops: embeds them, then reports leave-one-out kNN accuracy, a same/different-cat distance AUC, and a suggested confidence threshold. No model is built or promoted here. The progress, cancel, and history below are shared with Build gallery and Identify — one job runs at a time; the others queue behind it.
