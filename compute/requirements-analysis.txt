# Heavy ML deps for offline analysis (motion-gate validation oracles).
# **Install manually into .venv-compute:** `pip install -r compute/requirements-analysis.txt`
# The always-on collector (ingest, frame storage) runs without these; import only inside
# analysis backends (analysis/yolo.py, analysis/bsuv.py) so the hot ingest path stays lean.
#
# GPU (Windows / NVIDIA) — READ FIRST, there is a footgun:
#   A plain `pip install -r ...` pulls the CPU-only torch from PyPI, and it stays CPU-only:
#   ultralytics -> torchvision, and PyPI's torchvision pins the CPU torch, so it will
#   silently clobber any CUDA torch you installed. It must be a MATCHED pair from the CUDA
#   index, installed together. On Blackwell cards (e.g. RTX 5060 Ti, sm_120) the wheel must
#   be cu128 or newer — older CUDA wheels lack Blackwell kernels. Recipe:
#     pip install torch torchvision --index-url https://download.pytorch.org/whl/cu128
#     pip install ultralytics
#   Verify: python -c "import torch; print(torch.cuda.is_available())"  # must be True
#   (bump cu128 -> cu129/cu130 as newer CUDA wheels land).
